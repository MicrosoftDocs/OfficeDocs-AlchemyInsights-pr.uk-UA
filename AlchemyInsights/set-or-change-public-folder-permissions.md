# <a name="permissions-and-public-folders"></a>Дозволи та спільні папки

Ви можете змінити дозволи для спільних папок за допомогою програми Outlook, центру адміністрування Exchange (EAC) або PowerShell:
  
- Щоб отримати вказівки з Outlook, [клацніть тут](https://support.office.com/article/Set-or-change-permissions-for-a-public-folder-b2e0440c-7873-48ec-9ff2-b1a20b723005.aspx).
    
- У [цій статті](https://technet.microsoft.com/library/jj651147%28v=exchg.150%29.aspx.aspx#Anchor_1) наведено вказівки з указівок для EAC. 
    
- У PowerShell, зверніться до [цієї статті](https://technet.microsoft.com/library/bb124743%28v=exchg.160%29.aspx.aspx) , щоб отримати вказівки з використання надбудови чином. Якщо вам потрібні інструкції для підключення до Exchange PowerShell, клацніть [тут](https://technet.microsoft.com/library/jj984289%28v=exchg.160%29.aspx.aspx).
    
Якщо **зовнішні користувачі не можуть надсилати повідомлення електронної пошти до спільної папки з підтримкою пошти**, причиною може бути те, що спільна папка не містить дозволів, потрібних для зовнішньої доставки електронної пошти. Цю проблему можна вирішити за [допомогою інструкцій](https://support.microsoft.com/help/2984402/-5.7.1-smtp-550-5.7.1-resolver.rst.authrequired-nondelivery-report-when-external-users-try-to-send-mail-to-mail-enabled-public-folders-in-office-365.aspx)програми [Outlook, а також інструкцій PowerShell](https://technet.microsoft.com/library/aa997560%28v=exchg.150%29.aspx.aspx#Anchor_1).
